জাভাস্ক্রিপ্টে Var, Let এবং Const এর মধ্যে পার্থক্য

    Var: Var হল জাভাস্ক্রিপ্টে একটি ভেরিয়েবল ঘোষণা করার জন্য পুরনো কীওয়ার্ড। গ্লোবাল স্কোপড বা ফাংশন স্কোপড। var কীওয়ার্ডের স্কোপ হল গ্লোবাল বা ফাংশন স্কোপ। এর অর্থ হল ফাংশনের বাইরে সংজ্ঞায়িত ভেরিয়েবলগুলি গ্লোবাল অ্যাক্সেস করা যেতে পারে এবং একটি নির্দিষ্ট ফাংশনের ভিতরে সংজ্ঞায়িত ভেরিয়েবলগুলি ফাংশনের মধ্যে অ্যাক্সেস করা যেতে পারে।

    Let: Let কীওয়ার্ডটি Var কীওয়ার্ডের একটি উন্নত সংস্করণ। একটি Let ভেরিয়েবলের সুযোগ শুধুমাত্র ব্লক স্কোপড। এটি নির্দিষ্ট ব্লকের ({block}) বাইরে অ্যাক্সেসযোগ্য হতে পারে না।

    Const: Const কীওয়ার্ডের সমস্ত বৈশিষ্ট্য যা let কীওয়ার্ডের মতো, ব্যবহারকারী cosnt একবার ডিক্লিয়ার করলে পরবর্তীতে আপডেট করা যায় না যখন ব্যবহারকারীরা একটি কনস্ট ভেরিয়েবল ঘোষণা করে, তাদের এটি শুরু করতে হবে, অন্যথায়, এটি একটি ত্রুটি প্রদান করে। ব্যবহারকারী কনস্ট ভেরিয়েবলটি ঘোষণা করার পরে আপডেট করতে পারে না।

    Arrow Function এবং Regular Function মধ্যে পার্থক্য

    Regular Function ফাংশনগুলি গঠনযোগ্য, সেগুলিকে নতুন কীওয়ার্ড ব্যবহার করে কল করা যেতে পারে। যাইহোক, Arrow Function শুধুমাত্র কলযোগ্য এবং গঠনযোগ্য নয়, যেমন Arrow Function কখনই কনস্ট্রাক্টর ফাংশন হিসাবে ব্যবহার করা যাবে না। অতএব, তাদের কখনই নতুন কীওয়ার্ড দিয়ে Call করা যাবে না।

    Map(), forEach, filter, find এর মধ্যে পার্থক্য

    Map() objects হল key-value Pairs Functtion. Java Script এ Map() পদ্ধতিটি Present অ্যারেতে উপস্থিত প্রতিটি উপাদানের একটি নির্দিষ্ট Function Call করে একটি অ্যারে তৈরি করে। এটি একটি non-mutating পদ্ধতি। সাধারণত Map() পদ্ধতিটি অ্যারের প্রতিটি উপাদানে একটি অ্যারে এবং Calling Fanction এর উপর পুনরাবৃত্তি করতে ব্যবহৃত হয়।

    forEach() দ্বারা প্রক্রিয়াকৃত Provided পরিসর callback Function-এর প্রথম Invoked আগে সেট করা হয়। যে উপাদানগুলি ইতিমধ্যে পরিদর্শন করা হয়েছে, সেগুলি Callback Function দ্বারা পরিদর্শন করা হবে না ৷ যদি অ্যারের বিদ্যমান উপাদানগুলি পরিবর্তন করা হয় বা মুছে ফেলা হয়, তাহলে forEach() তাদের পরিদর্শনের সময় Callback Function-এ পাস করা তাদের মান হবে; যদি ইতিমধ্যেই পরিদর্শন করা উপাদানগুলিকে সরিয়ে দেওয়া হয় (যেমন shift() ব্যবহার করে) পুনরাবৃত্তির সময়, পরবর্তী উপাদানগুলি এড়িয়ে যাবে।

    filter() একটি অ্যারের প্রতিটি উপাদানের জন্য একটি প্রদত্ত Callback Function ফাংশনকে একবার কল করে এবং সমস্ত মানগুলির একটি নতুন অ্যারে তৈরি করে যার জন্য callback Function একটি মান ফেরত দেয় যা সত্যে বাধ্য করে। callback function শুধুমাত্র অ্যারের সূচীগুলির জন্য Invoked করা হয় যেগুলির মান নির্ধারণ করা আছে; এটি এমন সূচীগুলির জন্য Invoked করা হয় না যেগুলি মুছে ফেলা হয়েছে বা যাকে কখনও মান নির্ধারণ করা হয়নি। যে অ্যারে উপাদানগুলি Callback Function পরীক্ষায় উত্তীর্ণ হয় না সেগুলি এড়িয়ে যায় এবং নতুন অ্যারেতে অন্তর্ভুক্ত করা হয় না।

    find() function অ্যারের প্রতিটি সূচকের জন্য একবার callback function ফাংশনটি কার্যকর করে যতক্ষণ না callback function একটি সত্য মান প্রদান করে। যদি সেই উপাদানটির মান মিথ্যা হয় তাহলে undifine রিটার্ন করে দেয় callbackFn অ্যারের প্রতিটি সূচকের জন্য Invoked করা হয়, শুধুমাত্র নির্ধারিত মানগুলির জন্য নয়। এর মানে হল যে পদ্ধতিগুলি শুধুমাত্র নির্ধারিত মানগুলি পরিদর্শন করে তার তুলনায় এটি স্পার্স অ্যারেগুলির জন্য কম দক্ষ হতে পারে। find() function অ্যারেকে মিউটেট করে না যার উপর এটি কল করা হয়, তবে Callback function-এ দেওয়া ফাংশনটি করতে পারে।

    Why use Template Literals in Javascript ?

    Template literals হল ব্যাকটিক (`) Key দিয়ে সীমাবদ্ধ করা Characters, যা বহু-লাইন স্ট্রিং String Embedded Expressions Interpolation এবং Tag করা টেমপ্লেট নামক বিশেষ নির্মাণের জন্য অনুমতি দেয়। সাধারণ String থাকার পাশাপাশি, Template Literals এ Placeholder নামক অন্যান্য অংশও থাকতে পারে, যেগুলি dollar sign এবং curly braces:{ } দ্বারা সীমাবদ্ধ embedded expressions ${expression}।

    Template Literals কখনও কখনও অনানুষ্ঠানিকভাবে টেমপ্লেট স্ট্রিং বলা হয়, কারণ এগুলি সাধারণত String Interpolation এর জন্য ব্যবহৃত হয় যেতে পারে আক্ষরিক টেমপ্লেটের বিভিন্ন অংশে প্রায় সবই রাখা যায়
